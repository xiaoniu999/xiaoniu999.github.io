1. 关键字public protected private

称为访问修饰符，类成员包括成员变量和成员函数。类外部的定义是：不在类定义中的部分。

1.1 public

  1. 可以被类内外部访问；
  2. 继承后，在派生类中也是public；

1.2 protected

  1. 不能被类外部访问，比如实例化的对象不能访问protected中的成员；
  2. 可以在派生类中访问，在派生类中仍然是public；

1.3 private

  1. 只能在当前类中使用；

2. 继承方式

继承时可以选择继承方式：

  1. public 继承：基类 public 成员，protected 成员，private 成员的访问属性在派生类中分别变成：public, protected, private
  2. protected 继承：基类 public 成员，protected 成员，private 成员的访问属性在派生类中分别变成：protected, protected, private
  3. private 继承：基类 public 成员，protected 成员，private 成员的访问属性在派生类中分别变成：private, private, private

3. 构造函数和析构函数

在 C++ 中，类需要编写自定义构造函数和析构函数的情况通常包括但不限于以下几点：

3.1 自定义构造函数的情况

  1. 初始化资源：当类需要管理资源，如动态内存、文件句柄、网络连接、数据库连接等，需要在对象创建时进行初始化。
  2. 设置初始状态：为对象的成员变量提供初始状态，确保对象在创建后处于有效且一致的状态。
  3. 验证输入参数：在构造函数中检查传入的参数是否有效，如果参数不符合预期，可以抛出异常。
  4. 继承：如果类继承自一个或多个基类，可能需要在派生类的构造函数中调用基类的构造函数以确保正确的初始化顺序。
  5. 执行复杂的初始化逻辑：当对象的初始化需要执行一些复杂的逻辑或算法时。
  6. 多态：如果类设计为支持多态，可能需要在构造函数中进行一些特殊的设置，以确保对象能以正确的形式被使用。

3.2 自定义析构函数的情况

  1. 释放资源：当类管理了动态分配的内存、文件句柄、网络连接等资源，需要在对象生命周期结束时释放这些资源以避免资源泄漏。
  2. 执行清理操作：如果类在运行时改变了某些状态或数据，析构函数可以用来恢复到原始状态或执行其他清理工作。
  3. 防止资源泄漏：确保在对象被销毁前，所有分配的资源都得到妥善处理。
  4. 处理继承：如果一个类继承自其他类，并且基类有析构函数，派生类的析构函数需要调用基类的析构函数以确保资源被正确释放。
  5. 完成复杂的清理逻辑：当需要执行一些复杂的清理逻辑，如撤销注册、注销事件监听器等。

在这个例子中，Resource 类管理了一些资源，因此需要自定义构造函数来初始化这些资源，以及自定义析构函数来释放它们。如果不这样做，资源可能不会被正确释放，从而导致资源泄漏。

编写自定义的构造函数和析构函数是 C++ 语言中进行资源管理和对象生命周期管理的重要机制。正确使用构造函数和析构函数对于编写健壮且无内存泄漏的程序至关重要。

4. 虚函数

虚函数是 C++ 中面向对象编程的一个核心特性，它允许实现运行时多态（也称为动态多态或晚期多态）。运行时多态是指程序在运行时根据对象的实际类型来决定调用哪个函数的行为。

4.1 虚函数的定义

4.2 虚函数的作用

  1. 实现多态：虚函数允许基类指针或引用调用派生类中重写（override）的虚函数，这是多态的基础。
  2. 动态绑定：C++ 编译器在运行时根据对象的实际类型来解析虚函数调用，这个过程称为动态绑定或晚期绑定。
  3. 接口定义：虚函数常用于定义接口，确保任何派生类都有实现特定行为的能力。

4.6 虚函数的注意事项

  • 虚函数必须在基类中首先声明为 virtual，然后才能在派生类中被重写。
  • 如果基类中的虚函数被声明为 virtual，则所有派生类中的同名函数都会自动是虚函数。
  • 虚函数的调用解析是在运行时进行的，这与内联函数或静态函数的早期绑定（编译时解析）不同。
  • 虚函数使得代码更加灵活，但也可能会带来一些性能开销，因为运行时需要进行额外的类型检查。

虚函数是实现多态的关键，它允许开发者编写出更加通用和灵活的代码。

4.7.1 对于没有虚函数的类，是否能够使用基类指针管理派生类对象？

答案：不行。虽然基类指针仍然能够指向派生类对象，但是无法通过基类指针来调用派生类特有的成员函数。这是因为在没有虚函数的情况下，编译器会将基类指针视为只能访问基类中定义的成员，而不会考虑派生类中可能存在的同名成员。最根本的原因是，基类和派生类类型不同。这被称为静态绑定或早绑定，因为函数调用的解析是在编译时确定的。

在这个例子中，尽管derivedObj是Derived类的一个实例，basePtr指向它，但由于show()函数在Base类中没有被声明为virtual，编译器会在编译时决定调用哪个show()函数。由于basePtr是Base*类型的，编译器将调用Base类中的show()函数。

4.7.2 如果派生类继承了具有虚函数的基类，又增加了成员，还能用基类指针管理派生类对象吗?

答案：能。因为内存布局：派生类对象的内存布局以基类开始，随后是派生类特有的成员。这意味着派生类对象包含基类的所有成员，因此基类指针可以访问这些成员。
